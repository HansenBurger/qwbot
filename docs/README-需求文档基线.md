# 需求文档基线模板

> 本文档为需求文档的标准格式和内容指南，用于指导各章节的填写方式。  
> 所有需求文档应遵循此模板结构，确保信息完整性和可追溯性。

---

## 文档命名规范

```
docs/QWB-{编号}-{简短标题}.md
```

- **编号**: 三位数字，从 001 开始递增（如 QWB-001）
- **标题**: 使用中文简短描述，避免特殊字符（如 `消息模板优化`）
- **示例**: `QWB-001-消息模板优化.md`

---

## 文档结构

### 1. 需求背景

**填写指南：**
- 简要说明当前存在的问题或痛点
- 描述为什么需要这个改造
- 可以用 1-3 个具体场景说明现状不足
- 避免技术细节，聚焦业务影响

**示例：**
```
原有消息模板功能仅支持单一默认模板，编辑体验存在以下问题：
1. 编辑模板时需要先手动复制变量名，容易出错
2. 变量管理面板层级过深，不够直观
3. 移动端适配不完善，交互体验差
4. 缺少模板预览功能，无法直观查看最终效果
```

---

### 2. 改造范围

**填写指南：**
- 将需求拆分为多个独立的改造点
- 每个改造点用二级标题（`###`）单独说明
- 结构：实现逻辑 → 关键设计 → 文件改动

**每个改造点应包含：**

#### 实现逻辑
- 说明技术方案的核心思路
- 描述数据流向或交互流程
- 重点解释"怎么做"和"为什么这样做"

#### 关键设计（可选）
- 说明重要的设计决策
- 解释为什么选择某种方案而非其他方案
- 涉及权衡取舍时特别说明

#### 文件改动
- 列出修改的核心文件
- 每个文件用一行简述改动内容
- 格式：`- 文件名: 改动说明`

**示例：**
```
### 1. 多模板支持

**实现逻辑：**
- 新增 `reminder_templates` 表存储多个模板
- 首次运行时自动迁移旧数据
- 支持启用/禁用/删除操作

**关键设计：**
- 保留最后一个模板不允许删除，避免误操作

**文件改动：**
- `store.py`: 新增表和 CRUD 函数
- `web.py`: 新增模板管理路由
```

---

### 3. 技术细节

**填写指南：**
- 说明实现过程中的关键技术点
- 用三级标题（`####`）分点说明
- 包含代码片段或配置示例（如有）
- 重点解释容易出错或需要特别注意的地方

**示例：**
```
#### 数据迁移

首次访问时自动执行：
1. 检查新表是否为空
2. 读取旧字段内容
3. 创建默认记录

#### 拼音转换

使用 `pinyin-pro@3.26.0` 库：
- 支持多音字自动选择
- 转换为 snake_case 格式
```

---

### 4. 影响范围

**填写指南：**
- 列出受影响的系统模块
- 说明数据库变更（新增表/字段/索引）
- 说明 API 变更（新增/修改/删除接口）
- 说明兼容性问题（如有）

**示例：**
```
- 数据库：新增 `reminder_templates` 表
- 前端：重构消息模板管理区域
- API：新增模板 CRUD 接口
- 兼容性：旧版字段保留用于回退
```

---

### 5. 测试要点

**填写指南：**
- 列出需要验证的核心功能
- 每个测试点用一句话描述
- 覆盖正常流程和异常场景
- 按优先级排序（最重要的在前）

**示例：**
```
1. 多模板切换：启用/禁用/删除后自动启用下一个
2. 变量唯一性：重复变量名或链接地址时提示错误
3. 拼音转换：中文描述自动生成正确的 snake_case 变量名
4. 预览渲染：变量替换和 markdown 格式正确显示
5. 移动端：折叠面板、按钮布局、弹窗尺寸适配
```

## 写作原则

### 内容要求

1. **聚焦改造点**: 重点说明"做了什么"和"怎么做的"，而非"为什么做"
2. **技术导向**: 面向开发人员和后续维护者，而非业务用户
3. **代码精简**: 只在必要时贴代码片段，避免大段复制
4. **逻辑清晰**: 按功能模块组织，而非按文件组织

### 格式要求

1. **标题层级**: 
   - `#` 文档标题
   - `##` 大章节（需求背景、改造范围等）
   - `###` 改造点或子章节
   - `####` 技术细节

2. **列表使用**:
   - 无序列表（`-`）用于并列说明
   - 有序列表（`1.`）用于步骤或优先级

3. **代码标记**:
   - 文件名用反引号：`` `store.py` ``
   - 变量名用反引号：`` `var_name` ``
   - 短代码用反引号：`` `SELECT * FROM` ``

4. **表格使用**（可选）:
   - 对比前后差异时可用表格
   - 列出文件改动时可用表格

- [ ] 文档格式符合规范
