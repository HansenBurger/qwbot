# QWB-001 消息模板优化

> **需求编号**: QWB-001
> **创建日期**: 2026-07-22
> **状态**: 已完成

---

## 1. 需求背景

原有消息模板功能仅支持单一默认模板，编辑体验和变量管理存在以下问题：

1. 编辑模板时需要先手动复制变量名，容易出错，且在编辑链接时保存会导致之前的编辑内容丢失
2. 变量管理面板层级过深，系统变量和文档变量全部平铺展示，界面臃肿
3. 文档链接变量需要手动命名，缺少自动拼音转换能力
4. 移动端适配不完善，交互体验差
5. 缺少模板预览功能，无法直观查看最终推送效果

---

## 2. 改造范围

### 1. 可编辑消息模板

**实现逻辑：**
- 在 `store.py` 中定义 `DEFAULT_REMINDER_TEMPLATE` 和 `REMINDER_TEMPLATE_VARS` 常量
- 使用 `app_settings` 表存储用户自定义模板内容（`get_reminder_template` / `set_reminder_template`）
- 重构 `build_scheduled_reminder` 函数，支持模板占位符渲染（`{var_name}` 格式）
- 前端添加模板编辑面板，点击变量标签可插入占位符

**关键设计：**
- 模板存储在 `app_settings` 表中，支持恢复到默认模板
- 变量标签点击即插入，降低手动输入出错概率

**文件改动：**
- `qwbot/store.py`: 新增模板常量和存取函数
- `qwbot/message.py`: 重构消息构建逻辑，支持模板渲染
- `qwbot/web.py`: 新增 `/reminder-template` POST 路由
- `qwbot/templates/_notification_section.html`: 新增模板编辑器面板
- `qwbot/static/style.css`: 新增模板编辑器样式
- `qwbot/static/app.js`: 变量标签点击插入逻辑

---

### 2. 自定义模板变量管理（DB 化）

**实现逻辑：**
- 新增 `reminder_template_vars` 表，存储自定义变量（`var_name`, `var_label`, `var_value`）
- 首次运行时从 `.env` 配置中读取默认变量（如 `progress_doc_link`, `case_assignment_doc_link`）并初始化
- 自定义变量的 `var_value` 非空时覆盖 `.env` 值；为空时回退到 `.env` URL
- `build_scheduled_reminder` 合并系统变量 + DB 自定义变量
- 前端提供内联表格 UI 进行编辑，弹窗创建新变量

**关键设计：**
- DB 化管理无需重启服务即可生效，相比之前硬编码在 `.env` 中更灵活
- 变量分为「系统变量」（不可删除）和「自定义变量」（可 CRUD）

**文件改动：**
- `qwbot/store.py`: 新增 `reminder_template_vars` 表和 CRUD 函数
- `qwbot/message.py`: 合并系统变量和自定义变量
- `qwbot/web.py`: 新增变量管理路由
- `qwbot/templates/_notification_section.html`: 变量管理表格和弹窗
- `qwbot/static/style.css`: 表格和弹窗样式
- `qwbot/static/app.js`: 变量管理交互逻辑

### 3. 变量名自动拼音转换

**实现逻辑：**
- 创建变量弹窗中，字段顺序调整为：描述 → 变量名 → 链接地址
- 监听描述输入框的 `blur` 事件，当变量名为空时自动生成 snake_case 格式的变量名
- 转换规则：中文转拼音，空格/连字符转下划线，全小写，去除非法字符
- 文档链接变量固定搭配 `_doc_link` 后缀

**关键设计：**
- 仅在变量名为空时自动填充，用户仍可手动编辑
- 使用 `blur` 事件而非 `input` 事件触发，避免在输入法组合过程中意外触发（解决中文输入法兼容问题）

**文件改动：**
- `qwbot/static/app.js`: 新增 snake_case 格式化函数和 blur 事件监听
- `qwbot/static/style.css`: 字段提示样式
- `qwbot/templates/_notification_section.html`: 字段顺序调整

## 3. 技术细节

#### 模板渲染流程

1. 调用 `build_scheduled_reminder` 时，先加载当前模板内容
2. 合并系统变量（固定）和 DB 自定义变量
3. 遍历模板中的 `{var_name}` 占位符，用变量值替换
4. 文档链接变量渲染为企业微信 markdown 链接格式：`[描述](链接地址)`

#### 拼音转换兼容性

- 使用前端 JS 实现拼音转换（`pinyin-pro` 库）
- 绑定 `blur` 事件而非 `input` 事件，避免中文输入法 IME 组合阶段误触发
- 多音字使用默认读音

#### 变量优先级

- 自定义变量的 `var_value` 非空时，覆盖 `.env` 中的同名变量
- 自定义变量的 `var_value` 为空时，回退到 `.env` 中的 URL
- 系统变量（如 `{title}`, `{content}`）不可被覆盖

## 4. 影响范围

- **数据库**: 新增 `reminder_template_vars` 表；`app_settings` 表新增 `reminder_template` 记录
- **前端**: 重构消息模板管理区域，新增变量管理表格、创建弹窗、模板编辑器
- **API**: 新增模板保存路由、变量 CRUD 路由、测试发送路由
- **部署**: Dockerfile 依赖变更，部署脚本子命令重构
- **兼容性**: 旧版 `.env` 中的文档链接变量自动迁移到 DB，无需手动操作

## 5. 测试要点

1. 模板编辑：修改模板内容后保存，刷新页面确认持久化生效
2. 变量插入：点击变量标签确认占位符正确插入到光标位置
3. 变量管理：新增/编辑/删除自定义变量后，模板渲染结果正确更新
4. 拼音转换：输入中文描述后切换到链接输入框，变量名自动生成为 snake_case 格式
5. 测试发送：点击测试发送按钮确认消息正确推送到测试 webhook
6. 默认模板恢复：点击恢复默认按钮确认模板内容重置
7. 移动端适配：在窄屏下确认弹窗双列布局、按钮对齐、折叠面板交互正常
8. 部署脚本：各子命令（start/stop/restart/logs/status）均可正常执行
